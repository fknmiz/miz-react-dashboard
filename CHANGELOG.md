# Changelog

All notable changes to this project are documented here.

## [1.0.0] - 2026-05-12

### Released
- Published **miz-react-dashboard** as a polished frontend showcase for KPI storytelling, chart composition, and responsive dashboard systems.
- Positioned the repo as a UI craft artifact with product sensibility, not just a component gallery.
- Tightened the narrative around how business dashboards should feel when they are built for decisioning rather than decoration.

### Why this mattered
- Frontend portfolios often show isolated widgets but not a believable information hierarchy.
- Product and hiring teams need to see taste, restraint, and system thinking in the same repo.
- This release made the project readable as a serious UI foundation piece.

## [0.1.0] - 2026-02-01

### Shipped
- Locked the first cohesive layout system for cards, charts, sections, and theme behavior.
- Established the initial responsive grid and visual language that the public repo still builds on.

## [Prototype] - 2025-06-06

### Built
- Built the earliest dashboard prototype to test density, readability, and chart-to-summary balance.
- Validated that the repo could feel like a product surface rather than a style exercise.

## [Design Phase] - 2024-02-05

### Designed
- Focused on hierarchy, legibility, and believable KPI storytelling.
- Chose reusable patterns that could scale into other operator surfaces.
- Avoided ornamental complexity that would hurt scanability.

## [Idea Origin] - 2023-04-12

### Observed
- The idea started from a simple gap in many frontend portfolios: lots of components, not enough product judgment.
- The missing artifact was a dashboard that felt intentional at the system level.
